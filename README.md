Что это за репозиторий

АБИС — проект студенческой команды, который выполняется по заказу библиотеки учебного заведения. Здесь команда постепенно собирает автоматизированную систему учета книговыдачи: от исследования предметной области и требований до полноценного веб-приложения.

>Важно: репозиторий устроен как история развития проекта по этапам. Документы лежат по типам артефактов, а не "по этапам". Для навигации по контрольным срезам есть отдельные индексные файлы со ссылками на актуальные документы.

## О чем проект

Проект посвящен автоматизации учета книговыдачи в библиотеке.

## Основные функции:

· Учет книжного фонда (каталог с обложками)
· Учет читателей (студенты, сотрудники)
· Выдача и возврат книг
· Автоматический контроль просрочек
· Отчет «Просрочки» с фильтрацией
· Экспорт данных в CSV
· Аутентификация и разграничение ролей (читатель, библиотекарь, администратор)
· Личный кабинет читателя

Правила ведения документации

· каждый важный документ начинается с шапки: версия, статус, дата создания, дата обновления
· после шапки идут блоки «О документе», «Для кого» и «Основано на»
· каждый документ получает стабильный код вида DOC-AREA-NNN
· новые версии опираются на предыдущие, а не переписывают историю
· индекс контрольной точки хранит список артефактов и ссылок на них
· решения из поздних этапов ссылаются на ранние артефакты
· изменения трактовки методики, формул или границ MVP фиксируются явно
